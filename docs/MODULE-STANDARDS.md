# BIOL 1113 — Module Standards

How to build, structure, and name every module in this course.
Read this before creating or editing any week folder.

---

## File Structure

Each `weekNN/` folder contains exactly three HTML source files:

| File | Canvas page | Purpose |
|------|-------------|---------|
| `overview.html` | Week N Overview | Objectives + task list — where students start |
| `materials.html` | Week N Materials | Videos, readings, focus questions |
| `assignments.html` | Week N Assignments | ELQ reminder, HW, writing assignments, test info |

No other HTML files belong in week folders. Slides live in `slides/weekNN/`.

---

## Naming Conventions

- Folders: `week01` through `week15` (zero-padded)
- Files: exactly `overview.html`, `materials.html`, `assignments.html`
- Canvas page slugs are auto-generated by `upload_to_canvas.py` as `biol1113-weekNN-pagename`

---

## Content Rules

### Readings
- Always link to specific OpenStax Biology 2e sections, never the chapter root
- Never assign full chapters
- Include a one-sentence note on what to look for in each reading
- Readings use the `.reading-card` component

### Videos
- Each instructor video gets its own `.video-card`
- Title = descriptive topic name, not "Video 1" or "Lecture 3"
- Include 2–3 focus questions per video (pre-watch priming, not the ELQ questions)
- Runtime listed in video-meta
- ELQ questions are embedded in the video itself — do not list them on the materials page

### Objectives
- 4–6 measurable objectives per week
- Use action verbs: explain, describe, compare, predict, distinguish, apply, trace, analyze
- Avoid "understand" and "know" — they are not measurable
- Objectives go on `overview.html` only

### Task List (overview.html)
- Tasks appear in the order students should complete them: videos first, then readings, then assignments
- Each task links directly to the relevant page or Canvas item
- HW and test tasks only appear on weeks where those items exist

### Due Dates
- All work due Sunday 11:59 PM
- Use the `.due-banner` component on `assignments.html`
- Never say "by the end of the week" — state the specific date

### Supplemental Resources
- Maximum 1–2 per week
- External curated content only (not instructor-created)
- Marked clearly as optional
- Keep the section short — do not let it compete with required content

## CSS Components

All styles come from `course-styles.css`. Never write inline styles manually or add `<style>` blocks.

| Component | Class | Use for |
|-----------|-------|---------|
| Page header | `.page-header` | Top of every page |
| Week label | `.week-label` | Inside page header |
| Objectives | `.objectives-box` | Objectives on overview pages |
| Task list | `.task-list` | What-to-do list on overview |
| Video card | `.video-card` | Each video on materials pages |
| Reading card | `.reading-card` | Each reading on materials pages |
| Assignment block | `.assignment-block` | Each assignment on assignment pages |
| Due date banner | `.due-banner` | Top of assignments pages |
| Notice box | `.notice-box` | General info callouts |
| Points badge | `.points-badge` | Inside assignment block h3 |

## Build and Upload Workflow

1. Edit source HTML files in `weekNN/` — these preview correctly in a browser
2. Edit `course-styles.css` for any style changes
3. Run `bash build/build.sh` to generate Canvas-ready HTML in `build/biol1113/`
4. Run `python3 scripts/upload_to_canvas.py` to push to Canvas (uploads as drafts)
5. Review and publish in Canvas

Never edit files in `build/biol1113/` — they are overwritten on every build.

## What Belongs in the Repo vs. Canvas

| Item | Repo | Canvas |
|------|------|--------|
| HTML page content | ✅ source | ✅ uploaded via script |
| CSS | ✅ course-styles.css | inlined at build time |
| Slide decks (.key/.pptx) | ✅ slides/ | upload manually |
| Videos | ❌ too large | ✅ Kaltura/Canvas |
| ELQ questions | ❌ gitignored | ✅ Canvas quizzes |
| Test questions | ❌ gitignored | ✅ Canvas quizzes |
| Student grades/data | ❌ never | ✅ Canvas gradebook only |
| Canvas API token | ❌ .env only | N/A |
